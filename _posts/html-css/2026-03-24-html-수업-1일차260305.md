첫날 수업은 웹 시스템이 어떻게 구축되는지, 
그리고 그 밑바탕이 되는 네트워크와 HTML의 기초에 대해 다뤄보았다.</p>
<h2 id="1-웹-개발의-전체-흐름">1. 웹 개발의 전체 흐름</h2>
<p>웹 시스템을 구축하는 과정은 크게 화면을 만드는 프론트엔드와 데이터를 처리하는 백엔드로 나뉜다. </p>
<p>프론트엔드: 사용자가 보는 화면 개발 (HTML, CSS, JS, React.js, Vue.js 등) </p>
<p>백엔드(서버): 서버 로직 및 데이터 처리 (Java Spring Framework, Node.js, Python 등) </p>
<p>데이터베이스(DB): 티켓 예매 정보와 같은 서버 쪽 데이터를 저장 (SQL, MySQL, Oracle) </p>
<h2 id="2-컴퓨터-통신의-기초">2. 컴퓨터 통신의 기초</h2>
<p>컴퓨터가 서로 통신하기 위해서는 일정한 규칙인 프로토콜(Protocol)과 주소인 IP가 필요하다.</p>
<h4 id="ip-주소">IP 주소:</h4>
<p>32비트 체계(IPv4)로 구성되며, 공인 IP와 사설 IP로 나뉜다.</p>
<h4 id="공유기와-nat">공유기와 NAT:</h4>
<p>하나의 공인 IP를 받아 사설 IP(가상 IP)를 증폭시켜 여러 장치가 공유하게 해주는 기술이다.. </p>
<h4 id="dns-domain-name-server">DNS (Domain Name Server):</h4>
<p>'naver.com' 같은 도메인을 실제 IP 주소로 변환해 주는 역할을 한다. </p>
<h4 id="전송-프로토콜">전송 프로토콜:</h4>
<p>-TCP: 연결 지향형으로 전송 성공 여부(ACK)를 확인 (예: 웹사이트, 카카오톡) </p>
<p>-UDP: 비연결 지향형으로 속도가 빠르다. (예: 라이브 방송, 실시간 게임) </p>
<h4 id="httphtml">HTTP/HTML:</h4>
<p>문서를 전송하기 위한 하이레벨 프로토콜(HTTP)과 문서를 저장하는 규격(HTML)이다. </p>
<h2 id="3-웹의-시각적-요소">3. 웹의 시각적 요소</h2>
<h4 id="rgb">RGB:</h4>
<p>빛의 3원색을 0~255 단계로 표현하며, 웹에서는 주로 16진수 코드로 색상을 지정한다. </p>
<h4 id="이미지의-구조">이미지의 구조:</h4>
<p>사진은 수많은 점(Pixel)으로 구성되며, 데이터 소모를 줄이기 위해 코덱(Codec)으로 압축한다. </p>
<h2 id="4-html-실습">4. HTML 실습</h2>
<p>수업에서 직접 작성해 본 HTML의 기본 구조와 주요 태그들 실습</p>
<pre><code>&lt;!DOCTYPE html&gt;
&lt;html&gt;
&lt;head&gt;
    &lt;title&gt;문서 제목&lt;/title&gt;
&lt;/head&gt;
&lt;body&gt;
    &lt;h1&gt;제목 태그 (h1~h6)&lt;/h1&gt;
    &lt;p&gt;문단을 표시하는 태그입니다.&lt;/p&gt;
    &lt;a href=&quot;https://www.naver.com&quot;&gt;네이버 링크&lt;/a&gt;
    &lt;br&gt; &lt;img src=&quot;images/picture.jpg&quot; alt=&quot;풍경사진&quot;&gt;
&lt;/body&gt;
&lt;/html&gt;</code></pre><blockquote>
<p>경로 설정 팁: </p>
</blockquote>
<ul>
<li>절대 경로 (/images/...): 서버 루트 기준 경로로 보다 안전하게 사용 가능</li>
<li>상대 경로 (images/...): 현재 파일 위치 기준 경로</li>
<li>index.html: 서버 접속 시 폴더명까지만 입력해도 우선순위로 열리는 기본 파일명</li>
</ul>
