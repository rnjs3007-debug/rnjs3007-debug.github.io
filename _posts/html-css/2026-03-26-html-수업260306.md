<p>이전 수업의 복습내용을 포함한 새로운 HTML기본 틀에 대해 다뤄보았다.</p>
<h2 id="1-네트워크">1. 네트워크</h2>
<p>웹페이지를 만들기 전에 먼저 이해해야 기본 내용들</p>
<h3 id="가-주소와-이름-ip--dns">가. 주소와 이름 (IP &amp; DNS)</h3>
<p>IP 주소: 컴퓨터의 실제 주소</p>
<p>IPv4: 32비트 체계 (예: 255.255.255.255). 우리가 흔히 보는 형태</p>
<p>IPv6: 주소 고갈 문제를 해결하기 위한 128비트 체계</p>
<p>127.0.0.1: '로컬 호스트', 즉 내 컴퓨터 자신을 가리키는 약속된 번호</p>
<p>사설 IP: 192.168... 등으로 시작하며 공유기 내부에서 할당하는 가짜 주소</p>
<p>DNS (Domain Name System): naver.com 같은 문자를 IP 주소로 바꿔주는 전화번호부 역할</p>
<h3 id="나-통로와-규칙-port--protocol">나. 통로와 규칙 (Port &amp; Protocol)</h3>
<p>PORT (포트): 한 IP 안에서 실행되는 여러 프로그램 중 어느 방으로 갈지 정하는 문 번호 (1~65536)</p>
<p>HTTP (80): 일반적인 웹 통신</p>
<p>HTTPS (443): 암호화가 추가된 보안 통신</p>
<p>프로토콜: 통신 규칙</p>
<p>TCP: 꼼꼼하게 확인하며 데이터를 주고받음 (웹, 채팅).</p>
<p>UDP: 확인 없이 빠르게 보냄 (스트리밍, 게임).</p>
<p>쿼리 파라미터 (Query Parameter): 주소창 뒤에 붙는 ?query=검색어 형태
서버에 특정 데이터를 요청할 때 사용함</p>
<h2 id="2-html-기본-구조와-텍스트-태그">2. HTML 기본 구조와 텍스트 태그</h2>
