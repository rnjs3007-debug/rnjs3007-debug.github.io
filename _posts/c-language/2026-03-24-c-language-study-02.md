## 1. 프로그램의 기초 및 필수 사항 


시작점: 모든 C 프로그램은 main 함수를 최우선 순위로 시작한다. 


헤더 파일: 표준 입출력을 위해 #include <stdio.h>(standard input output)가 필수이며, 이를 통해 각종 함수를 불러올 수 있다. 

## 2. 연산자 (Operator) 


기본 개념: 예를 들어 2+3에서 2와 3은 연산을 당하는 피연산자이며, 계산은 왼쪽에서 오른쪽 순서로 진행된다. 


연산자 우선순위: '증산시 관비 논삼대' 순서를 따른다. 
>증감(++, --) > 산술(+, -) > 시프트(<<) > 관계(>) > 비트(&, |) > 논리(&&) > 삼항 > 대입(=)

#### *주요 연산자:


%: 나눗셈의 나머지를 구하는 나머지 연산자 


++ / --: 값을 1씩 증가시키거나 감소 


**->후증감 주의사항: printf("%d", a++);와 같이 사용하면 출력을 먼저 수행한 후에 값을 증감시킨다.(출력물 값에는 영향을 끼치지 않음!) **

```
int a = 1;
printf("%d\n", ++a); // (1)
printf("%d\n", ++a); // (2)
printf("%d\n", a);   // (3)
printf("%d\n", a--); // (4)
printf("%d\n", a);   // (5)
```


## 3. 비트 연산자 (Bit Operator) 

#### -진법 이해:

2진수(BIN): 0, 1로 구성 

8진수: 0~7 

16진수(HEX): 0~9, A~F (예: AF = 175) 
```
#include <stdio.h>

int main() {
    int num = 175;

    printf("대문자 출력: %X\n", num);  // AF
    printf("소문자 출력: %x\n", num);  // af
    printf("0x 포함 출력: %0x\n", num); // 0xAF (16진수임을 명시함)

    return 0;
}
```

#### -비트 이동(Shift):텍스트


```
<< (왼쪽 시프트): 값을 왼쪽으로 밀어내며, 결과적으로 2배씩 곱하는 것과 동일한 효과 
(숫자 커지는 개념)


>> (오른쪽 시프트): 값을 오른쪽으로 밀어내며 2로 나누는 효과
(숫자 작아지는 개념)
```
 
예시: int a=5; int b=a<<2; 결과값은 20(2진수 00010100)이 된다. 


#### -논리 비트 연산:


| (OR): 비트 단위의 덧셈 개념 

& (AND): 비트 단위의 곱하기 개념 

^ (XOR): 두 비트가 서로 다르면 1, 같으면 0 

~ (NOT): 비트를 완전히 반전시키는 경우 (예: 00000001 → 11111110)


## 3. 형식 지정자 (Format Specifier)

정수형:** %d (10진수)**, %o (8진수), %x (16진수)

실수형: **%f (float/double**), %e (지수 형태)

문자 및 문자열: **%c (단일 문자**), %s (문자열)

진법 표기: 16진수를 대문자로 출력하려면 %X를 사용


## 4. 변수와 메모리

-주소 연산자 (&): 변수 앞에 &를 붙이면 해당 변수가 저장된 메모리의 시작 주소를 의미. scanf 함수에서 값을 입력받을 때 주로 사용.

-데이터 크기: sizeof 연산자를 사용하여 자료형이나 변수의 메모리 크기(byte)를 확인할 수 있다.

## 5. 주요 비트 연산 활용 

XOR (^): 두 수의 비트가 서로 다를 때만 1을 반환하며, 암호화나 두 변수의 값을 맞바꿀 때 유용하게 쓰인다.

NOT (~): 1의 보수를 만드는 연산으로, 부호가 있는 정수에서는 -(n+1)의 결과가 나온다.

## 6. 하드웨어 아키텍처와 엔디언 (Endianness)

### -저장 방식의 차이:

x86 (Little Endian): 하위 바이트(LSB)를 앞 주소에 먼저 저장 

ARM (Big Endian): 상위 바이트(MSB)를 앞 주소에 먼저 저장

### -메모리:
1바이트는 8비트이며, 아키텍처에 따라 비트 저장 순서가 다를 수 있다.

### -비트 맞추기: 

서로 다른 아키텍처 간에 데이터를 정확히 전달하기 위해 시프트(<<, >>)와 OR(|) 연산을 사용하여 바이트 순서를 뒤집는 작업이 필요하다. 

>
문제: x86(리틀 엔디언)에서 만든 short a = 3; 데이터를 ARM(빅 엔디언)이 읽을 수 있게 비트 연산으로 변환하는 과정은? 


```
#include <stdio.h>

int main(void) {
    // 1. x86 방식(리틀 엔디언)으로 저장된 숫자 3
    // 실제 메모리 모습: [00000011][00000000] (오른쪽 신발이 앞에 있음)
    short a = 3; 

    // 2. 뒤에 처박혀 있던 '진짜 3' 덩어리를 앞으로 8칸 밀기 (오른쪽으로 시프트)
    // 결과: [00000000][00000011]
    short b = a >> 8; 

    // 3. 앞에 잘못 서 있던 덩어리를 뒤로 8칸 밀기 (왼쪽으로 시프트)
    // 결과: [00000000][00000000] (이 예제에서는 원래 앞이 0이라서 0)
    short c = a << 8; 

    // 4. 두 조각을 합체(OR 연산)해서 정상적인 모양 만들기
    // 결과: [00000000][00000011] (드디어 ARM이 읽을 수 있는 진짜 '3' 완성!)
    short d = b | c; 

    printf("변환 전(x86 기준): %d\n", a);
    printf("변환 후(ARM 기준): %d\n", d);

    return 0;
}
```


>문제: char a = 1; char b = ~a; 일 때 b의 값은? 

풀이:

a는 00000001


~(NOT 연산)은 비트를 완전 반전시킨다.

결과는 11111110이며, 이는 10진수로 254이다.


## 7. 음수 표현 방식

-부호 비트: 8비트(char) 기준 가장 왼쪽 비트(MSB)가 1이면 음수를 의미 

-2의 보수 방식: 비트를 반전시킨 후 1을 더하여 음수를 표현

-MSB (Most Significant Bit): 8비트 중 가장 왼쪽 비트. 이게 0이면 양수, 1이면 음수

### -음수 만드는 법: 
반전(NOT) 시키고 +1 하기.

### 중요한 이유:
컴퓨터가 뺄셈을 덧셈처럼 처리하여 회로를 단순하게 만들기 위함/ 시스템 프로그래밍이나 임베디드(하드웨어 제어)를 할 때 비트 하나 차이로 숫자가 양수에서 음수로 확 변하는 '오버플로우(Overflow)' 현상을 이해해야함


```
#include <stdio.h>

int main(void) {
    char a = -1; 
    
    // %d로 출력하면 우리가 아는 '-1'로 보여줍니다.
    printf("10진수 출력: %d\n", a); 
    
    // %u(부호 없는 정수)로 출력하면 컴퓨터 내부의 실제 값인 '255'가 보입니다.
    printf("컴퓨터 내부 값(미부호): %u\n", (unsigned char)a); 
    
    return 0;
}
```
>*** 음수를 만드는 3단계 공정 (2의 보수)
->숫자 -1을 8비트(char) 기준으로 만드는 과정
1단계: 양수 상태
+1을 2진수로 씁니다. → 0000 0001
2단계: 비트 반전 (1의 보수)
0은 1로, 1은 0으로 모두 뒤집습니다. → 1111 1110
3단계: 1 더하기 (2의 보수 - 완성!)
반전된 값에 1을 더합니다. → 1111 1111


## 8. 마무리 실습문제 정리하기

### 1. 성적 산출 (범위 체크 및 프로그램 종료)
가장 중요한 점은 잘못된 값이 들어왔을 때 프로그램을 더 진행시키지 않고 바로 종료하는 것.

```
#include <stdio.h>
#include <stdlib.h> // exit() 함수 사용을 위해 필수
#pragma warning(disable:4996) // scanf 오류 방지

int main(void) {
    int num;
    printf("점수 입력(0~100): ");
    scanf("%d", &num);

    // 1. 예외 처리 (범위 밖의 숫자)
    if (num < 0 || num > 100) {
        printf("오류: 0에서 100 사이의 숫자만 입력하세요.\n");
        exit(1); 
    }

    // 2. 등급 판별
    if (num >= 90)      printf("A\n");
    else if (num >= 80) printf("B\n");
    else if (num >= 70) printf("C\n");
    else if (num >= 60) printf("D\n");
    else                printf("E\n");

    return 0;
}
```

### 2. 세 수 중 최댓값 구하기 (알고리즘 최적화)
else if를 복잡하게 쓰는 것보다, max 변수를 활용하는 방식이 훨씬 직관적이고 확장성이 좋다.
```
int main(void) {
    int a, b, c, max;
    printf("세 개의 숫자 입력: ");
    scanf("%d %d %d", &a, &b, &c);

    max = a; // 일단 첫 번째 숫자가 가장 크다고 가정
    if (max < b) max = b; // b가 더 크면 max를 b로 교체
    if (max < c) max = c; // c가 더 크면 max를 c로 교체

    printf("가장 큰 수: %d\n", max);
    return 0;
}
```

>
### 요약
if ~ else if: 여러 선택지 중 딱 하나만 골라 실행하고 싶을 때 사용
연속 if: 조건이 맞다면 여러 개를 모두 확인하거나 실행해야 할 때 사용

### 3. 나이에 따른 요금 계산 및 3의 배수 판별
조건문의 논리 구조를 깔끔하게 배치하는 것이 핵심이다.
```
// 나이에 따른 요금 (adult: 1000, teen: 500, kid: free)
int main(void) {
    int age;
    printf("나이 입력 (0~130): ");
    scanf("%d", &age);

    if (age < 0 || age > 130) {
        printf("입력 오류\n");
        exit(1);
    }

    if (age >= 20)      printf("성인 요금: 1000원\n");
    else if (age >= 8)  printf("청소년 요금: 500원\n");
    else                printf("미취학 아동: 무료\n");

    return 0;
}

// 3의 배수 판별
int main(void) {
    int n;
    printf("숫자 입력: ");
    scanf("%d", &n);

    if (n % 3 == 0) printf("3의 배수입니다.\n");
    else            printf("3의 배수가 아닙니다.\n");

    return 0;
}
```
### 4. 절댓값 구하기 (데이터 직접 변경 방식)
출력만 하는 것이 아니라 변수 자체의 값을 양수로 바꿔주는 것이 데이터 처리 관점에서 더 확실하다.
```
int main(void) {
    int num;
    printf("숫자 입력: ");
    scanf("%d", &num);

    if (num < 0) {
        num = num * -1; // 음수라면 부호를 반전시켜 저장
    }

    printf("절댓값 결과: %d\n", num);
    return 0;
}
```

>중괄호({ })의 기본 규칙
실행할 문장이 1개일 때: 중괄호를 생략 가능
실행할 문장이 2개 이상일 때: 반드시 중괄호를 사용

>자료형,크기(Byte),크기(Bit),주요 용도
char, 1, 8, 문자 저장
short, 2, 16, 작은 정수
int, 4, 32, 일반적인 정수 (가장 많이 사용)
double, 8, 64, 정밀한 실수(소수점)
