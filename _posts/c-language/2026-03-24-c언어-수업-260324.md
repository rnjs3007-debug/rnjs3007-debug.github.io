<h2 id="1-프로그램의-기초-및-필수-사항">1. 프로그램의 기초 및 필수 사항</h2>
<p>시작점: 모든 C 프로그램은 main 함수를 최우선 순위로 시작한다. </p>
<p>헤더 파일: 표준 입출력을 위해 #include &lt;stdio.h&gt;(standard input output)가 필수이며, 이를 통해 각종 함수를 불러올 수 있다. </p>
<h2 id="2-연산자-operator">2. 연산자 (Operator)</h2>
<p>기본 개념: 예를 들어 2+3에서 2와 3은 연산을 당하는 피연산자이며, 계산은 왼쪽에서 오른쪽 순서로 진행된다. </p>
<p>연산자 우선순위: '증산시 관비 논삼대' 순서를 따른다. </p>
<blockquote>
<p>증감(++, --) &gt; 산술(+, -) &gt; 시프트(&lt;&lt;) &gt; 관계(&gt;) &gt; 비트(&amp;, |) &gt; 논리(&amp;&amp;) &gt; 삼항 &gt; 대입(=)</p>
</blockquote>
<h4 id="주요-연산자">*주요 연산자:</h4>
<p>%: 나눗셈의 나머지를 구하는 나머지 연산자 </p>
<p>++ / --: 값을 1씩 증가시키거나 감소 </p>
<p>*<em>-&gt;후증감 주의사항: printf(&quot;%d&quot;, a++);와 같이 사용하면 출력을 먼저 수행한 후에 값을 증감시킨다.(출력물 값에는 영향을 끼치지 않음!) *</em></p>
<pre><code>int a = 1;
printf(&quot;%d\n&quot;, ++a); // (1)
printf(&quot;%d\n&quot;, ++a); // (2)
printf(&quot;%d\n&quot;, a);   // (3)
printf(&quot;%d\n&quot;, a--); // (4)
printf(&quot;%d\n&quot;, a);   // (5)</code></pre><h2 id="3-비트-연산자-bit-operator">3. 비트 연산자 (Bit Operator)</h2>
<h4 id="-진법-이해">-진법 이해:</h4>
<p>2진수(BIN): 0, 1로 구성 </p>
<p>8진수: 0~7 </p>
<p>16진수(HEX): 0<del>9, A</del>F (예: AF = 175) </p>
<pre><code>#include &lt;stdio.h&gt;

int main() {
    int num = 175;

    printf(&quot;대문자 출력: %X\n&quot;, num);  // AF
    printf(&quot;소문자 출력: %x\n&quot;, num);  // af
    printf(&quot;0x 포함 출력: %0x\n&quot;, num); // 0xAF (16진수임을 명시함)

    return 0;
}</code></pre><h4 id="-비트-이동shift텍스트">-비트 이동(Shift):텍스트</h4>
<pre><code>&lt;&lt; (왼쪽 시프트): 값을 왼쪽으로 밀어내며, 결과적으로 2배씩 곱하는 것과 동일한 효과 
(숫자 커지는 개념)


&gt;&gt; (오른쪽 시프트): 값을 오른쪽으로 밀어내며 2로 나누는 효과
(숫자 작아지는 개념)</code></pre><p>예시: int a=5; int b=a&lt;&lt;2; 결과값은 20(2진수 00010100)이 된다. </p>
<h4 id="-논리-비트-연산">-논리 비트 연산:</h4>
<p>| (OR): 비트 단위의 덧셈 개념 </p>
<p>&amp; (AND): 비트 단위의 곱하기 개념 </p>
<p>^ (XOR): 두 비트가 서로 다르면 1, 같으면 0 </p>
<p>~ (NOT): 비트를 완전히 반전시키는 경우 (예: 00000001 → 11111110)</p>
<h2 id="3-형식-지정자-format-specifier">3. 형식 지정자 (Format Specifier)</h2>
<p>정수형:** %d (10진수)**, %o (8진수), %x (16진수)</p>
<p>실수형: <strong>%f (float/double</strong>), %e (지수 형태)</p>
<p>문자 및 문자열: <strong>%c (단일 문자</strong>), %s (문자열)</p>
<p>진법 표기: 16진수를 대문자로 출력하려면 %X를 사용</p>
<h2 id="4-변수와-메모리">4. 변수와 메모리</h2>
<p>-주소 연산자 (&amp;): 변수 앞에 &amp;를 붙이면 해당 변수가 저장된 메모리의 시작 주소를 의미. scanf 함수에서 값을 입력받을 때 주로 사용.</p>
<p>-데이터 크기: sizeof 연산자를 사용하여 자료형이나 변수의 메모리 크기(byte)를 확인할 수 있다.</p>
<h2 id="5-주요-비트-연산-활용">5. 주요 비트 연산 활용</h2>
<p>XOR (^): 두 수의 비트가 서로 다를 때만 1을 반환하며, 암호화나 두 변수의 값을 맞바꿀 때 유용하게 쓰인다.</p>
<p>NOT (~): 1의 보수를 만드는 연산으로, 부호가 있는 정수에서는 -(n+1)의 결과가 나온다.</p>
<h2 id="6-하드웨어-아키텍처와-엔디언-endianness">6. 하드웨어 아키텍처와 엔디언 (Endianness)</h2>
<h3 id="-저장-방식의-차이">-저장 방식의 차이:</h3>
<p>x86 (Little Endian): 하위 바이트(LSB)를 앞 주소에 먼저 저장 </p>
<p>ARM (Big Endian): 상위 바이트(MSB)를 앞 주소에 먼저 저장</p>
<h3 id="-메모리">-메모리:</h3>
<p>1바이트는 8비트이며, 아키텍처에 따라 비트 저장 순서가 다를 수 있다.</p>
<h3 id="-비트-맞추기">-비트 맞추기:</h3>
<p>서로 다른 아키텍처 간에 데이터를 정확히 전달하기 위해 시프트(&lt;&lt;, &gt;&gt;)와 OR(|) 연산을 사용하여 바이트 순서를 뒤집는 작업이 필요하다. </p>
<blockquote>
</blockquote>
<p>문제: x86(리틀 엔디언)에서 만든 short a = 3; 데이터를 ARM(빅 엔디언)이 읽을 수 있게 비트 연산으로 변환하는 과정은? </p>
<pre><code>#include &lt;stdio.h&gt;

int main(void) {
    // 1. x86 방식(리틀 엔디언)으로 저장된 숫자 3
    // 실제 메모리 모습: [00000011][00000000] (오른쪽 신발이 앞에 있음)
    short a = 3; 

    // 2. 뒤에 처박혀 있던 '진짜 3' 덩어리를 앞으로 8칸 밀기 (오른쪽으로 시프트)
    // 결과: [00000000][00000011]
    short b = a &gt;&gt; 8; 

    // 3. 앞에 잘못 서 있던 덩어리를 뒤로 8칸 밀기 (왼쪽으로 시프트)
    // 결과: [00000000][00000000] (이 예제에서는 원래 앞이 0이라서 0)
    short c = a &lt;&lt; 8; 

    // 4. 두 조각을 합체(OR 연산)해서 정상적인 모양 만들기
    // 결과: [00000000][00000011] (드디어 ARM이 읽을 수 있는 진짜 '3' 완성!)
    short d = b | c; 

    printf(&quot;변환 전(x86 기준): %d\n&quot;, a);
    printf(&quot;변환 후(ARM 기준): %d\n&quot;, d);

    return 0;
}</code></pre><blockquote>
<p>문제: char a = 1; char b = ~a; 일 때 b의 값은? </p>
</blockquote>
<p>풀이:</p>
<p>a는 00000001</p>
<p>~(NOT 연산)은 비트를 완전 반전시킨다.</p>
<p>결과는 11111110이며, 이는 10진수로 254이다.</p>
<h2 id="7-음수-표현-방식">7. 음수 표현 방식</h2>
<p>-부호 비트: 8비트(char) 기준 가장 왼쪽 비트(MSB)가 1이면 음수를 의미 </p>
<p>-2의 보수 방식: 비트를 반전시킨 후 1을 더하여 음수를 표현</p>
<p>-MSB (Most Significant Bit): 8비트 중 가장 왼쪽 비트. 이게 0이면 양수, 1이면 음수</p>
<h3 id="-음수-만드는-법">-음수 만드는 법:</h3>
<p>반전(NOT) 시키고 +1 하기.</p>
<h3 id="중요한-이유">중요한 이유:</h3>
<p>컴퓨터가 뺄셈을 덧셈처럼 처리하여 회로를 단순하게 만들기 위함/ 시스템 프로그래밍이나 임베디드(하드웨어 제어)를 할 때 비트 하나 차이로 숫자가 양수에서 음수로 확 변하는 '오버플로우(Overflow)' 현상을 이해해야함</p>
<pre><code>#include &lt;stdio.h&gt;

int main(void) {
    char a = -1; 

    // %d로 출력하면 우리가 아는 '-1'로 보여줍니다.
    printf(&quot;10진수 출력: %d\n&quot;, a); 

    // %u(부호 없는 정수)로 출력하면 컴퓨터 내부의 실제 값인 '255'가 보입니다.
    printf(&quot;컴퓨터 내부 값(미부호): %u\n&quot;, (unsigned char)a); 

    return 0;
}</code></pre><blockquote>
<p>*** 음수를 만드는 3단계 공정 (2의 보수)
-&gt;숫자 -1을 8비트(char) 기준으로 만드는 과정
1단계: 양수 상태
+1을 2진수로 씁니다. → 0000 0001
2단계: 비트 반전 (1의 보수)
0은 1로, 1은 0으로 모두 뒤집습니다. → 1111 1110
3단계: 1 더하기 (2의 보수 - 완성!)
반전된 값에 1을 더합니다. → 1111 1111</p>
</blockquote>
<h2 id="8-마무리-실습문제-정리하기">8. 마무리 실습문제 정리하기</h2>
<h3 id="1-성적-산출-범위-체크-및-프로그램-종료">1. 성적 산출 (범위 체크 및 프로그램 종료)</h3>
<p>가장 중요한 점은 잘못된 값이 들어왔을 때 프로그램을 더 진행시키지 않고 바로 종료하는 것.</p>
<pre><code>#include &lt;stdio.h&gt;
#include &lt;stdlib.h&gt; // exit() 함수 사용을 위해 필수
#pragma warning(disable:4996) // scanf 오류 방지

int main(void) {
    int num;
    printf(&quot;점수 입력(0~100): &quot;);
    scanf(&quot;%d&quot;, &amp;num);

    // 1. 예외 처리 (범위 밖의 숫자)
    if (num &lt; 0 || num &gt; 100) {
        printf(&quot;오류: 0에서 100 사이의 숫자만 입력하세요.\n&quot;);
        exit(1); 
    }

    // 2. 등급 판별
    if (num &gt;= 90)      printf(&quot;A\n&quot;);
    else if (num &gt;= 80) printf(&quot;B\n&quot;);
    else if (num &gt;= 70) printf(&quot;C\n&quot;);
    else if (num &gt;= 60) printf(&quot;D\n&quot;);
    else                printf(&quot;E\n&quot;);

    return 0;
}</code></pre><h3 id="2-세-수-중-최댓값-구하기-알고리즘-최적화">2. 세 수 중 최댓값 구하기 (알고리즘 최적화)</h3>
<p>else if를 복잡하게 쓰는 것보다, max 변수를 활용하는 방식이 훨씬 직관적이고 확장성이 좋다.</p>
<pre><code>int main(void) {
    int a, b, c, max;
    printf(&quot;세 개의 숫자 입력: &quot;);
    scanf(&quot;%d %d %d&quot;, &amp;a, &amp;b, &amp;c);

    max = a; // 일단 첫 번째 숫자가 가장 크다고 가정
    if (max &lt; b) max = b; // b가 더 크면 max를 b로 교체
    if (max &lt; c) max = c; // c가 더 크면 max를 c로 교체

    printf(&quot;가장 큰 수: %d\n&quot;, max);
    return 0;
}</code></pre><blockquote>
</blockquote>
<h3 id="요약">요약</h3>
<p>if ~ else if: 여러 선택지 중 딱 하나만 골라 실행하고 싶을 때 사용
연속 if: 조건이 맞다면 여러 개를 모두 확인하거나 실행해야 할 때 사용</p>
<h3 id="3-나이에-따른-요금-계산-및-3의-배수-판별">3. 나이에 따른 요금 계산 및 3의 배수 판별</h3>
<p>조건문의 논리 구조를 깔끔하게 배치하는 것이 핵심이다.</p>
<pre><code>// 나이에 따른 요금 (adult: 1000, teen: 500, kid: free)
int main(void) {
    int age;
    printf(&quot;나이 입력 (0~130): &quot;);
    scanf(&quot;%d&quot;, &amp;age);

    if (age &lt; 0 || age &gt; 130) {
        printf(&quot;입력 오류\n&quot;);
        exit(1);
    }

    if (age &gt;= 20)      printf(&quot;성인 요금: 1000원\n&quot;);
    else if (age &gt;= 8)  printf(&quot;청소년 요금: 500원\n&quot;);

    return 0;
}

// 3의 배수 판별
int main(void) {
    int n;
    printf(&quot;숫자 입력: &quot;);
    scanf(&quot;%d&quot;, &amp;n);

    if (n % 3 == 0) printf(&quot;3의 배수입니다.\n&quot;);
    else            printf(&quot;3의 배수가 아닙니다.\n&quot;);

    return 0;

<p>출력만 하는 것이 아니라 변수 자체의 값을 양수로 바꿔주는 것이 데이터 처리 관점에서 더 확실하다.</p>
<pre><code>int main(void) {
    int num;
    printf(&quot;숫자 입력: &quot;);
    scanf(&quot;%d&quot;, &amp;num);

    if (num &lt; 0) {
        num = num * -1; // 음수라면 부호를 반전시켜 저장
    }

    printf(&quot;절댓값 결과: %d\n&quot;, num);
    return 0;
}</code></pre><blockquote>

실행할 문장이 2개 이상일 때: 반드시 중괄호를 사용</p>
</blockquote>
