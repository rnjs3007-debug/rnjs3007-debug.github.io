<h2 id="과제-내용">과제 내용</h2>
<p>김소월의 시 <strong>진달래꽃</strong> 텍스트에서 <strong>4번 이상 등장한 글자</strong>를 출력하는 프로그램을 작성하라.</p>
<hr />
<h2 id="전체-코드">전체 코드</h2>
<pre><code class="language-python">text = &quot;&quot;&quot;나보기가 역겨워 가실때에는 말없이 고이 보내 드리오리다.
영변에 약산 진달래꽃 아름따다 가실 길에 뿌리오리다
가시는 걸을 걸음 놓인 그 꽃을 사뿐히 즈려 밟고 가시옵소서
나 보기가 역겨워 가실때에는 죽어도 아니 눈물 흘리오리다.&quot;&quot;&quot;

# char   = character(문자)
# count  = 횟수
# letter = 글자

char_count = {}  # 빈 딕셔너리

for letter in text:            # text를 한 글자씩 읽기
    if letter in char_count:
        char_count[letter] += 1    # 이미 있으면 +1
    else:
        char_count[letter] = 1     # 처음이면 1로 추가

for letter in char_count:
    count = char_count[letter]     # 횟수를 변수에 따로 저장

    if count &gt;= 4 and letter != &quot; &quot;: #파이썬이랑 비슷하게 작성해봄. 주로사용하는 패턴;
        print(letter, &quot;:&quot;, count)</code></pre>
<hr />
<h2 id="코드-흐름-설명">코드 흐름 설명</h2>
<h3 id="1단계--빈-딕셔너리-선언">1단계 — 빈 딕셔너리 선언</h3>
<pre><code class="language-python">char_count = {}</code></pre>
<ul>
<li><code>{}</code> 는 빈 딕셔너리를 만드는 문법이다.</li>
<li>나중에 <code>{ 글자: 횟수 }</code> 형태로 채워질 예정이다.</li>
<li>예시: <code>{'가': 8, '이': 6, '시': 5, ...}</code></li>
</ul>
<hr />
<h3 id="2단계--글자별-횟수-세기">2단계 — 글자별 횟수 세기</h3>
<pre><code class="language-python">for letter in text:
    if letter in char_count:
        char_count[letter] += 1
    else:
        char_count[letter] = 1</code></pre>
<ul>
<li><code>for letter in text</code> : text를 한 글자씩 꺼내서 <code>letter</code> 에 담는다.</li>
<li><code>if letter in char_count</code> : 이 글자가 딕셔너리에 이미 있는지 확인한다.</li>
<li>있으면 → 기존 횟수에 1을 더한다 (<code>+= 1</code>)</li>
<li>없으면 → 새로 추가하고 횟수를 1로 설정한다</li>
</ul>
<p>실제로 진행되는 과정을 따라가면 이렇다.</p>
<pre><code>text = &quot;나나보&quot; 라고 가정하면

첫 번째: letter = '나'
  '나' in char_count? → False (비어있음)
  → char_count = {'나': 1}

두 번째: letter = '나'
  '나' in char_count? → True (있음)
  → char_count = {'나': 2}

세 번째: letter = '보'
  '보' in char_count? → False (처음 등장)
  → char_count = {'나': 2, '보': 1}</code></pre><hr />
<h3 id="3단계--조건에-맞는-글자만-출력">3단계 — 조건에 맞는 글자만 출력</h3>
<pre><code class="language-python">for letter in char_count:
    count = char_count[letter]

    if count &gt;= 4 and letter != &quot; &quot;:
        print(letter, &quot;:&quot;, count)</code></pre>
<ul>
<li><code>for letter in char_count</code> : 딕셔너리를 순회하면 <strong>키(글자)</strong> 가 하나씩 나온다.</li>
<li><code>count = char_count[letter]</code> : 해당 글자의 횟수를 <code>count</code> 변수에 따로 저장한다. (가독성을 위해)</li>
<li>조건 1 — <code>count &gt;= 4</code> : 횟수가 4번 이상인가?</li>
<li>조건 2 — <code>letter != &quot; &quot;</code> : 공백이 아닌가?</li>
<li><code>and</code> 로 연결했으므로 <strong>두 조건 모두 만족해야</strong> 출력된다.</li>
</ul>
<blockquote>
<p>💡 공백은 등장 횟수가 매우 많지만 분석에 의미 없는 문자이므로 제외했다.</p>
</blockquote>

<h2 id="헷갈렸던-부분--in-키워드가-왜-두-가지로-해석되는가">헷갈렸던 부분 —&gt; <code>in</code> 키워드가 왜 두 가지로 해석되는가?</h2>
<p>코드를 짜면서 가장 헷갈렸던 부분이 바로 <code>in</code> 이었다.<br />같은 <code>in</code> 인데 상황마다 해석이 달랐기 때문이다.</p>
<hr />
<h3 id="1-for-letter-in-text-의-in">1. <code>for letter in text</code> 의 <code>in</code></h3>
<pre><code class="language-python">for letter in text:</code></pre>
<ul>
<li><code>for</code> 와 함께 쓰이는 <code>in</code> 이다.</li>
<li><strong>&quot;text에서 하나씩 꺼내서 letter에 담아라&quot;</strong> 라는 의미다.</li>
<li><strong>반복(순회)</strong> 의 역할을 한다.</li>
</ul>
<hr />
<h3 id="2-if-letter-in-char_count-의-in">2. <code>if letter in char_count</code> 의 <code>in</code></h3>
<pre><code class="language-python">if letter in char_count:</code></pre>
<ul>
<li><code>if</code> 와 함께 쓰이는 <code>in</code> 이다.</li>
<li><strong>&quot;letter가 char_count 안에 있는가?&quot;</strong> 라는 의미다.</li>
<li><strong>확인(검색)</strong> 의 역할을 한다.</li>
<li>결과는 <code>True</code> 또는 <code>False</code> 둘 중 하나다.</li>
</ul>

<h3 id="비유로-이해하기">비유로 이해하기</h3>
<pre><code>for letter in text
→ 책(text)에서 글자를 한 장씩 꺼내는 것

if letter in char_count
→ 서랍(char_count) 안에 그 글자가 있는지 확인하는 것</code></pre><hr />
<h3 id="헷갈리는-부분-내용-정리">헷갈리는 부분 내용 정리</h3>
<pre><code class="language-python">for __ in __:   # for가 앞에 → 꺼내서 반복
if __ in __:    # if가 앞에  → 있는지 확인</code></pre>
<blockquote>
<p>💡 <code>in</code> 자체는 같지만 <code>for</code> 와 함께 쓰이냐, <code>if</code> 와 함께 쓰이냐에 따라 다르게 작동한다.</p>
</blockquote>

<h2 id="이번-과제에서-배운-것">이번 과제에서 배운 것</h2>
<ul>
<li>딕셔너리는 <code>{ 키: 값 }</code> 구조로 데이터를 저장한다</li>
<li><code>if 변수 in 딕셔너리</code> 로 키가 존재하는지 확인할 수 있다</li>
<li><code>for 변수 in 딕셔너리</code> 로 순회하면 <strong>키</strong>가 하나씩 나온다</li>
<li><code>and</code> 로 조건 두 개를 동시에 걸 수 있다</li>
<li>변수명을 의미있게 짓는 것이 코드 가독성에 중요하다 (<code>ch</code> → <code>letter</code>, <code>count_dict</code> → <code>char_count</code>)</li>
</ul>
<hr />
<blockquote>
<p>딕셔너리를 배우자마자 바로 실전에 써본 과제! 처음엔 <code>in</code> 키워드가 상황마다 다르게 해석돼서 헷갈렸는데, <code>for</code> 와 함께 쓰면 꺼내기, <code>if</code> 와 함께 쓰면 확인하기라는 걸 이해하고 나니 훨씬 명확해졌다. 
두가지는 비교해서 숙지해두는 것으로 연습을 꾸준하게 해야겠다.</p>
</blockquote>
