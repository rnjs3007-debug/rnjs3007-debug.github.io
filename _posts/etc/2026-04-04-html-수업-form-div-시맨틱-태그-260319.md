<p>오늘은 데이터를 서버로 보내는 form 태그랑, 화면을 구역 나누는 div, 그리고 검색엔진이 좋아하는 시맨틱 태그까지 배웠다. 점점 HTML이 단순히 화면 만드는 게 아니라 데이터를 다루는 도구라는 게 느껴지는 수업이었다. 
아직은 구현하고픈 화면을 완벽하게 만들지는 못하지만, HTML 뼈대를 먼저 잡고 어떠한 구성으로 태그들이 이루어져 있는지에 대한 감이라는 것을 잡게 되는 것 같다.</p>
<hr />
<h2 id="1-form-태그--데이터-전송의-기본">1. form 태그 — 데이터 전송의 기본</h2>
<p>&quot;입력한 내용을 서버로 보내는 봉투&quot; 같은 역할이다.</p>
<pre><code class="language-html">&lt;form action=&quot;https://서버주소/&quot; method=&quot;post&quot;&gt;
    &lt;!-- 입력 요소들 --&gt;
&lt;/form&gt;</code></pre>
<ul>
<li><code>action</code> : 데이터를 받을 서버 주소</li>
<li><code>method=&quot;post&quot;</code> : 내용을 URL에 노출시키지 않고 몰래 보내는 방식 (로그인할 때 이걸 씀)</li>
<li><code>method=&quot;get&quot;</code> : URL에 데이터가 그대로 보임 (검색할 때 씀)</li>
</ul>
<hr />
<h2 id="2-input-종류들">2. input 종류들</h2>
<table>
<thead>
<tr>
<th>type</th>
<th>설명</th>
</tr>
</thead>
<tbody><tr>
<td><code>text</code></td>
<td>한 줄 텍스트 입력창</td>
</tr>
<tr>
<td><code>checkbox</code></td>
<td>여러 개 중 복수 선택 가능</td>
</tr>
<tr>
<td><code>radio</code></td>
<td>여러 개 중 하나만 선택 (name이 같아야 묶임)</td>
</tr>
<tr>
<td><code>submit</code></td>
<td>폼 데이터 전송 버튼</td>
</tr>
</tbody></table>
<pre><code class="language-html">&lt;input type=&quot;radio&quot; name=&quot;subject&quot; id=&quot;kor&quot; value=&quot;kor&quot;&gt;
&lt;label for=&quot;kor&quot;&gt;국어&lt;/label&gt;

&lt;input type=&quot;radio&quot; name=&quot;subject&quot; id=&quot;math&quot; value=&quot;math&quot;&gt;
&lt;label for=&quot;math&quot;&gt;수학&lt;/label&gt;</code></pre>
<p><strong>포인트:</strong> radio는 <code>name</code>이 같아야 하나의 그룹으로 묶인다. name이 다르면 둘 다 선택돼버린다.</p>
<hr />
<h2 id="3-label-태그--입력창이랑-글자-연결하기">3. label 태그 — 입력창이랑 글자 연결하기</h2>
<pre><code class="language-html">&lt;label for=&quot;user-name&quot;&gt;이름 입력:&lt;/label&gt;
&lt;input type=&quot;text&quot; id=&quot;user-name&quot; name=&quot;query&quot; required&gt;</code></pre>
<ul>
<li><code>for=&quot;아이디&quot;</code> 로 input이랑 연결하면, 글자 클릭해도 입력창에 커서가 생김</li>
<li><code>required</code> : 필수 항목. 비워두고 전송하면 브라우저가 경고 띄움</li>
<li>UX 면에서 label은 무조건 쓰는 게 좋음</li>
</ul>
<hr />
<h2 id="4-typesubmit--우체통에-넣기">4. type=&quot;submit&quot; — 우체통에 넣기</h2>
<pre><code class="language-html">&lt;input type=&quot;submit&quot; value=&quot;전송하기&quot;&gt;</code></pre>
<ul>
<li><code>&lt;form&gt;</code> 안에 담긴 모든 내용을 <code>action</code> 주소로 실제로 던지는 버튼</li>
<li>일반 버튼(<code>type=&quot;button&quot;</code>)은 눌러도 아무 일도 안 일어나지만, submit은 누르는 순간 데이터가 날아감</li>
<li>서류 작성 다 마치고 마지막에 찍는 <strong>'승인 도장'</strong> 같은 것</li>
</ul>
<hr />
<h2 id="5-div-태그--구역-나누기">5. div 태그 — 구역 나누기</h2>
<p>아무 의미 없는 투명한 박스. 관련된 요소들 묶어서 CSS 한 번에 적용하거나 위치 조정할 때 씀.</p>
<pre><code class="language-html">&lt;div&gt;
    &lt;label for=&quot;user-name&quot;&gt;이름 입력:&lt;/label&gt;
    &lt;input type=&quot;text&quot; id=&quot;user-name&quot; name=&quot;query&quot; required&gt;
&lt;/div&gt;</code></pre>
<ul>
<li><code>&lt;br&gt;</code> 남발하는 것보다 div에 <code>margin</code>/<code>padding</code> 주는 게 훨씬 깔끔하고 수정하기 편함</li>
<li><strong>Bootstrap</strong> : 다른 개발자들이 미리 만들어둔 CSS 모음집. 버튼, 메뉴바 등 가져다 쓰면 됨</li>
</ul>
<hr />
<h2 id="6-video-태그">6. video 태그</h2>
<pre><code class="language-html">&lt;video controls autoplay muted loop width=&quot;100%&quot; src=&quot;파일명.mp4&quot;&gt;&lt;/video&gt;</code></pre>
<ul>
<li><code>controls</code> : 재생 버튼, 볼륨 바 표시</li>
<li><code>autoplay muted</code> : 자동재생은 <strong>무조건 음소거 상태여야</strong> 브라우저가 허용함</li>
<li><code>loop</code> : 끝나면 처음부터 다시 재생</li>
<li><code>width=&quot;100%&quot;</code> : 부모 요소 너비에 맞게 꽉 채움</li>
</ul>
<hr />
<h2 id="7-시맨틱-태그--검색엔진이-좋아하는-구조">7. 시맨틱 태그 — 검색엔진이 좋아하는 구조</h2>
<p>단순히 div만 쓰는 것보다 의미 있는 태그를 써야 검색엔진이 구조를 잘 파악한다.</p>
<pre><code class="language-html">&lt;header&gt;머리 부분&lt;/header&gt;
&lt;main&gt;본문 부분&lt;/main&gt;
&lt;footer&gt;꼬리 부분&lt;/footer&gt;</code></pre>
<ul>
<li><code>&lt;meta name=&quot;description&quot; content=&quot;설명&quot;&gt;</code> : 검색 결과에 뜨는 사이트 설명 텍스트</li>
<li><strong>엔티티 코드</strong> : <code>&lt;</code> 같은 특수문자를 본문에 그대로 표시하고 싶을 때 사용<ul>
<li><code>&amp;lt;</code> → <code>&lt;</code></li>
<li><code>&amp;gt;</code> → <code>&gt;</code></li>
</ul>
</li>
</ul>
